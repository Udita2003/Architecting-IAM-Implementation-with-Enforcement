# IAM Implementation with Enforcement

## Project: Architecting IAM Implementation with Enforcement

## Overview
This project demonstrates the implementation of AWS IAM policies, roles, 
and enforcement using AWS Config and Lambda.

## Steps Completed
- ✅ Step 1: Evaluated IAM permissions using Access Control Matrix
- ✅ Step 2: Updated 4 IAM policies to match Access Control Matrix
- ✅ Step 3: Attached policies to appropriate IAM roles
- ✅ Step 4: Tested roles to verify correct permissions
- ✅ Step 5: Updated Lambda code to enforce resource restrictions
- ✅ Step 6: Marked bad policy as non-compliant in AWS Config
- ✅ Step 7: Created organizational role and policy visualization

## Standout Features
- ✅ Updated condition statements for more restrictive access
- ✅ Configured automated remediation in AWS Config
- ✅ Enabled SNS alerting in Lambda for non-compliant policies
- ✅ Created AWS Config monitoring flow diagram
